# Design Backlog

## 文档目的

本文档记录当前阶段暂不实现、但后续必须补充设计的功能。Leader 在开始每个里程碑设计前，应检查本文档，确认哪些设计项需要进入当前 `Mx_DESIGN.md`。

`docs/builder/Mx_DESIGN.md` 面向 Builder，只写实现所需的设计、边界、验收和测试内容。Leader 的工作汇报不写入 `Mx_DESIGN.md`，只在对产品经理的回复中说明。

Reviewer 需要独立的审查指南，放在 `docs/reviewer/`。审查指南只写评审目标、阅读顺序、风险清单、验证命令和通过标准，不重复 Builder 的完整实现说明。

## 使用规则

- 每次设计新的里程碑前，Leader 必须检查本文件。
- 如果某个待办项进入当前里程碑，应在对应 `docs/builder/Mx_DESIGN.md` 中写清楚技术方案、模块职责、实现边界、测试设计和验收标准。
- 每个里程碑完成 Builder 设计后，应补充对应 `docs/reviewer/Mx_REVIEW_GUIDE.md`。
- 如果某个待办项继续延期，应说明延期原因，并保持在本文件中。
- Builder 不应主动实现本文件中的待办项，除非当前 `Mx_DESIGN.md` 明确要求。
